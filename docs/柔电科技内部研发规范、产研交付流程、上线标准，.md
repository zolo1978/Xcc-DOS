# 一、整体前置说明
产研流程标准链路：
`需求评审 → 技术方案设计 → 开发 → 自测/联调 → 测试准入 → 集成测试 → 性能/安全测试 → 发布准入 → 灰度上线 → 全量上线 → 运维值守 → 复盘归档`
当前仅有 **产品PRD**，距离正式投产还差**一整套标准化配套文档**，下面按「优先级+模块」逐一拆解。

# 二、第一优先级：上线**强制必备文档**（不齐全无法提测/发布，P0）
## 1. 产品类补充文档（产品负责人编写）
### 1.1 需求清单 & 需求基线文档
**用途**：统一需求范围、验收标准、排期，杜绝需求变更，评审留痕。
**核心内容**
- 版本迭代编号、需求来源、业务背景、价值收益
- 功能清单（新增/修改/下线）、每个功能对应的 PRD 页码/模块
- 需求变更记录（变更人、时间、原因、影响范围）
- 上线范围：环境（测试/预发/生产）、租户/用户群体、灰度范围
**字节规范要求**：所有迭代必须锁定**需求基线**，变更走变更流程+留痕。

### 1.2 产品验收用例（产品+测试共建）
区别于测试用例，偏**业务视角验收**，是产品最终把关依据。
**核心内容**
- 按模块拆分场景、正常流程、异常流程、边界场景
- 预期结果、操作步骤、前置条件、后置影响
- 与PRD话术、交互、规则一一对齐

### 1.3 回滚/降级预案（产品牵头，产研测共同确认）
当前PRD仅简单提及回滚，字节要求**独立成文、可执行、可演练**。
**核心内容**
- 不同故障等级（P0/P1/P2）对应的降级策略、功能开关
- 手动回滚操作步骤、执行人员、执行顺序、影响范围
- 故障对外口径（用户提示语、客服话术、公告文案）
- 恢复验证标准、重启/切流流程

## 2. 技术类核心文档（后端/前端架构师/开发负责人编写）
### 2.1 总体技术方案（Tech Design Doc，TDD）
字节**核心强制文档**，架构/技术方案评审唯一依据。
**结构（字节标准模板）**
1. 业务背景 & 需求概述
2. 整体架构图（部署架构、网络架构、模块调用拓扑图）
3. 模块拆分、职责边界、依赖服务/中间件清单
4. 核心流程时序图、数据流图
5. 存储设计（数据库、缓存、文件、消息队列）
6. 核心算法/规则实现（Prolog 引擎调用逻辑、状态机实现、反哺流水线逻辑）
7. 兼容性说明（版本兼容、旧数据兼容、会话兼容）
8. 风险点 & 规避方案
9. 排期、责任人、分工

### 2.2 接口文档（API Doc）
前后端、跨服务联调必备，统一使用**接口规范 + 在线文档（如Apifox/OpenAPI/Swagger）**。
**字节规范要点**
- 统一请求方法、URL 命名规范、请求头（TraceID/SessionID/TenantID/Token）
- 统一入参、出参、数据类型、枚举、错误码体系
- 每个接口标注：功能说明、权限、限流规则、超时时间、重试策略
- 异常返回格式统一，与前端提示文案对齐
- 区分：对内接口、对外公网接口、内部服务接口

### 2.3 数据库设计文档（DB Design）
**强制独立文档**，DBA + 开发共同评审。
**核心内容**
1. 数据库选型、字符集、引擎、分库分表策略（多租户重点）
2. 全表结构：字段名、类型、长度、注释、默认值、索引、主键、外键
3. 索引设计（普通索引、联合索引、唯一索引）、索引使用场景
4. 数据字典、枚举值定义
5. 数据生命周期、清理策略、归档规则
6. 读写分离、缓存联动规则
7. 数据脱敏、加密规则（隐私字段）

### 2.4 部署架构 & 资源规划文档
用于运维部署、机器申请、集群规划。
**核心内容**
- 环境划分：开发/测试/预发/生产/压测/边缘环境
- 集群拓扑：节点角色、数量、机器配置（CPU/内存/磁盘）
- 网络架构：内网/公网隔离、端口开放策略、防火墙规则
- 中间件部署：Redis、MQ、LLM推理服务、Prolog进程部署拓扑
- 资源水位预估、扩容阈值、缩容规则

## 3. 测试类文档（测试负责人编写）
### 3.1 完整测试用例（功能+边界+异常+性能+安全）
字节测试标准：全覆盖场景，可自动化落地。
**分类**
- 功能测试用例：主流程、分支流程
- 边界测试：输入长度、会话超时、数据临界值
- 异常测试：服务宕机、网络中断、超时、重试、熔断
- 多租户隔离测试、并发测试
- 兼容性测试：浏览器、版本、历史数据

### 3.2 测试报告（准入/准出报告）
- 测试准入报告：提测时提交，说明版本、用例执行率、缺陷情况
- 上线准出报告：上线前最终报告，遗留缺陷说明、风险评估、测试结论

# 三、第二优先级：研发配套文档（开发过程必备，P1）
## 1. 前端专项文档
1. **页面路由 & 组件设计文档**
   页面结构、组件拆分、公共组件复用、样式规范、静态资源管理。
2. **前端埋点文档**
   字节全链路埋点规范：页面访问、按钮点击、会话行为、异常报错、兜底率指标埋点，用于数据分析&监控。
3. **前端兼容性适配文档**
   浏览器兼容、移动端适配、低版本终端兼容细节。

## 2. 后端专项文档
### 2.1 模块详细设计文档（LLD 低阶设计）
TDD 是高阶设计，LLD 面向**单模块编码**，细化函数、类、方法逻辑。
### 2.2 定时任务/流水线设计文档
针对「日志聚类、规则生成、定时清理、会话清理」等异步任务：
- 执行时机、执行周期、重试机制、失败告警、幂等性设计
### 2.3 缓存设计文档
Redis 键设计、过期策略、淘汰策略、缓存击穿/穿透/雪崩防护、缓存更新逻辑。

## 3. 规则引擎专项文档（本项目核心）
当前基于 Prolog 规则引擎，字节对**规则类系统**要求单独归档：
1. **规则编写规范**
   Prolog 语法约束、状态机编写标准、命名规范、禁止写法（死循环、无效规则）
2. **规则导入/导出/转换规范**
   Prolog 与备选引擎（Python/Node）格式互转规则、校验逻辑
3. **规则版本管理规范**
   版本号规则、快照存储、灰度规则、规则权限

## 4. 联调文档
跨团队/跨服务联调时间表、联调接口清单、问题跟踪表、联调结论。

# 四、第三优先级：运维、监控、安全、合规文档（上线&长期运行必备，P2）
## 1. 运维手册（运维编写，开发配合）
字节运维标准分为：**日常运维手册、应急运维手册、部署运维手册**
### 1.1 部署运维手册
- 编译打包流程、镜像构建规则、容器启动命令
- 环境变量配置、配置文件说明、配置中心使用规则
- 单机/集群/边缘设备三种部署步骤
- 启停、重启、进程查看、日志查看命令

### 1.2 日常运维手册
- 常规巡检项（进程、端口、连接数、磁盘、内存、CPU）
- 日志查看路径、日志级别切换
- 规则批量导入/清理、会话管理操作

### 1.3 应急运维手册（故障SOP）
**字节强要求：故障必须有标准化SOP**
- 故障分级、告警接收人、响应时效
- 常见故障现象、排查路径、一键处理命令
- 熔断开关、降级开关、流量切分操作步骤
- 重启/节点摘除/扩容缩容操作规范

## 2. 监控 & 告警文档
### 2.1 监控指标文档
梳理全链路监控指标，对接内部监控平台：
- 业务指标：请求量、成功率、兜底率、会话数、在线人数
- 性能指标：响应耗时、P95/P99耗时、并发数
- 系统指标：CPU、内存、磁盘、网络、连接数
- 错误指标：错误码分布、异常数、超时数

### 2.2 告警规则文档
- 告警阈值、告警级别、告警渠道（IM/电话/邮件）
- 告警收敛策略、夜间静默规则、告警升级机制

## 3. 安全专项文档（安全团队评审）
### 3.1 安全设计文档
- 鉴权体系、Token 设计、防重放、防篡改
- 接口限流、防刷、CC 防护规则
- 数据加密：传输加密（HTTPS）、存储加密（隐私字段）
- 注入防护、XSS 防护（前端）

### 3.2 安全测试报告
渗透测试、漏洞扫描结果、风险修复情况。

## 4. 合规 & 数据文档（法务/数据团队）
针对多租户、隐私数据场景（字节强合规）
1. **数据合规说明**：用户数据采集范围、使用范围、留存周期
2. **隐私政策配套说明**：脱敏、删除、导出流程
3. **多租户数据隔离合规说明**：不同租户数据隔离证明

# 五、第四优先级：迭代治理、归档、复盘文档（上线后/迭代收尾，P3）
## 1. 上线清单 & 上线checklist
字节发布标准：**上线前逐项勾选**，防止漏操作
- 配置检查、版本包检查、权限检查、流量检查、监控告警检查
- 灰度步骤、观察时长、全量上线确认项

## 2. 版本发布记录
每一次发布留痕：版本号、发布时间、发布人、变更内容、机器范围、问题记录。

## 3. 迭代复盘文档
迭代结束后产出：
- 进度回顾、需求达成情况
- 线上问题、缺陷分析、根因定位
- 优化项、后续改进计划、经验沉淀

## 4. 知识库归档
所有文档统一归档至内部知识库，按「产品/技术/测试/运维」分类，权限管控。

## 5. 培训文档（对内/对外）
- 内部培训：新同事上手文档、规则引擎使用培训
- 对外（如有）：租户使用手册、管理员操作手册
