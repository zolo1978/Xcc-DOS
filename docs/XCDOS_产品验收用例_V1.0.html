2. 进入驾驶舱

AC-DASH-003正常流程P1 驾驶舱支持下钻查看详情 AC-DASH-001 通过 1. 点击 KPI "异常数"

二、目标管理 (AC-GOAL-001 ~ 006)

编号类型优先级验收场景前置条件操作步骤预期结果
AC-GOAL-001正常流程P0 创建完整 SMART 目标 以 manager 角色登录，所在组织已存在 1. 进入 /goals
2. 点击"新建目标"
3. 填写名称/描述/指标类型/目标值/期限/负责人
4. 提交		 目标创建成功，列表可见，状态为 draft；字段完整无丢失
AC-GOAL-002正常流程P0 目标拆解为子目标 存在一个活跃目标（父目标） 1. 在父目标详情页
2. 点击"拆解子目标"
3. 创建子目标
4. 查看目标树		 子目标创建成功，parent_id 指向父目标，目标树正确展示父子关系
AC-GOAL-003正常流程P0 目标进度更新 存在状态为 active 的目标 1. 目标负责人登录
2. 进入目标详情
3. 更新 current_value		 进度条实时变化，百分比正确计算 (current/target*100)
AC-GOAL-004异常P0 创建目标时必填字段缺失 无 1. 进入新建目标
2. 不填名称/目标值/期限
3. 提交		 表单校验拦截，必填字段标红提示，不创建空数据
AC-GOAL-005边界P1 目标 deadline 为过去日期 无 1. 新建目标
2. deadline 选择过去日期
3. 提交		 允许创建但标注"已过期"状态，或前端给出二次确认提示
AC-GOAL-006权限P1 员工只能查看自己负责或关联的目标 存在 2 个不同 owner 的目标 1. employee A 登录
2. 进入目标列表		 仅显示 employee A 为目标负责人或参与人的目标，不显示他人目标

三、拆推评算中心 (AC-DEC-001 ~ 009)

编号类型优先级验收场景前置条件操作步骤预期结果
AC-DEC-001正常流程P0 从目标关联问题并录入 存在活跃目标 1. 进入目标详情
2. 点击"关联问题"
3. 填写问题标题/严重度/描述
4. 提交		 问题创建成功，goal_id 正确关联，状态 open
AC-DEC-002正常流程P0 完整拆推评算流程——拆解 存在问题状态=open 1. 问题详情 → 创建决策案例
2. 添加假设（≥2 条）
3. 为假设填写证据强度/置信度
4. 添加反例		 决策案例 stage=dismantle，假设列表完整展示
AC-DEC-003正常流程P0 完整拆推评算流程——推演 AC-DEC-002 通过，假设 ≥2 条 1. 基于假设选择推演方向
2. 生成推演方案
3. 确认推演结果		 案例阶段进入 hypothesize，生成 ≥1 个推演方案
AC-DEC-004正常流程P0 完整拆推评算流程——评估与试算 AC-DEC-003 通过 1. 进入评估阶段
2. 4 维度打分（资源/时间/风险/可行性）
3. 填写评估说明
4. ROI 试算（输入成本/收入/假设）		 案例阶段进入 evaluate→calculate；评估 4 维度分数+说明保存成功；ROI/回收期计算正确
AC-DEC-005正常流程P0 决策报告生成与导出 AC-DEC-004 通过 1. 进入报告阶段
2. 点击"生成报告"
3. 查看报告内容		 报告自动汇总：问题→假设→评估 4 维分→ROI→推演结论；报告可导出/分享
AC-DEC-006正常流程P1 决策案例状态流转 存在不同阶段的决策案例 1. 决策案例从 draft→in_progress→resolved→approved
2. 每一阶段检查前置条件		 状态流转符合规则：draft 不可直接跳到 approved；stage 和 status 联动正确
AC-DEC-007异常P0 未完成拆解就尝试评估 决策案例 stage=dismantle 1. 跳过假设登记
2. 直接点击"进入评估"		 系统拦截，提示"请先完成拆解阶段的假设登记"，不允许跳阶段
AC-DEC-008边界P1 假设反例为空时的置信度计算 假设登记中 counter_example 为空 1. 创建假设
2. 不填反例
3. 提交		 允许保存，但置信度评分受限（最高不超过 70）或提示补充反例
AC-DEC-009权限P1 非创建者不能修改他人决策案例 存在 user A 创建的决策案例 1. user B 登录（同角色）
2. 进入 user A 的决策案例详情
3. 尝试编辑/修改		 编辑按钮不可见或置灰，API 返回 403

AC-TASK-004边界P1 任务到期当天 23:59 前不算延期 任务 due_time 为今日 23:59 1. 今日 18:00 查看任务状态
2. 次日 00:01 查看任务状态		 18:00 时状态仍为 todo/in_progress（未延期）；次日 00:01 变为 delayed
AC-TASK-005异常P1 任务负责人被禁用后的任务处理 存在任务，其 owner 用户状态变为 inactive 1. 管理员禁用某用户
2. 查看该用户的任务		 任务列表中任务状态正常显示，可重新分配 owner，不丢失数据
