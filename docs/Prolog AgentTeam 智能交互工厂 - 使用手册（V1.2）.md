# Prolog AgentTeam 智能交互工厂 - 使用手册（V1.2）

## 一、快速开始

### 1.1 登录

1. 打开管理端 URL（由管理员提供）
2. 输入用户名、密码、租户编码
3. 首次登录后建议修改密码

### 1.2 界面概览

```
侧边导航
├── 租户管理（超管）
├── 用户管理
├── 规则配置 ← 核心
├── 同义词管理
├── 会话监控
├── 日志审计
└── 自进化任务
```

---

## 二、规则配置（核心操作）

### 2.1 新增规则

1. 进入 **规则配置** → 点击 **新增规则**
2. 填写：
   - **规则编码**：唯一标识，如 `RULE_SALES_001`
   - **规则名称**：中文描述，如"销售咨询意图识别"
   - **规则类型**：single（单规则）/ composite（组合规则）/ fallback（兜底）
   - **优先级**：数字越大越优先（建议 100 起）
   - **规则内容**：Prolog 语法编写（见规则编写规范）
3. 提交 → 规则状态为 **草稿**

### 2.2 规则灰度发布

1. 规则通过测试后 → 点击 **启用**
2. 设置灰度比例：建议 10% → 30% → 50% → 100%
3. 每个灰度阶段观察至少 4 小时
4. 灰度期间旧版本规则继续运行
5. 灰度期出现异常 → 灰度比例调回 0%

### 2.3 规则回滚

1. 进入规则详情 → **版本历史**
2. 选择要回滚到的历史版本
3. 当前版本自动保存快照 → 回滚生效

---

## 三、同义词管理

### 3.1 新增同义词

1. 进入 **同义词管理** → 新增
2. 标准词：如"退款"
3. 同义词：如"退钱"、"返款"、"退费"
4. 设置优先级（多条匹配时优先级高的返回）

### 3.2 停用同义词

- 状态切换为"停用" → 该同义词不再参与意图匹配

---

## 四、会话监控

### 4.1 活跃会话

- 查看当前所有活跃会话的：租户、用户、会话状态、开始时间、剩余有效时间

### 4.2 会话详情

- 点击某会话 → 查看上下文数据、最近交互记录、规则命中情况

---

## 五、日志审计

### 5.1 请求日志

- 查看每次 API 请求的：请求类型、规则命中、响应状态、耗时
- 支持按时间范围、会话、请求状态筛选

### 5.2 操作日志

- 查看所有管理端操作的：操作人、操作类型（增/删/改）、操作对象、变更前后值

---

## 六、自进化任务

### 6.1 规则聚类

- 系统自动识别相似规则 → 建议合并 → 人工审核 → 确认或拒绝

### 6.2 规则自动生成

- 基于标注样本 → AI 生成候选规则 → 人工审核 → 通过后上线

### 6.3 审核 AI 生成规则

1. 进入 **自进化任务** → 查看待审核任务
2. 检查 AI 生成的规则
3. 选择：通过（直接上线）/ 修改后通过 / 拒绝

---

## 七、常见问题

| 问题 | 答案 |
|------|------|
| 规则上线后不生效？ | 检查规则状态是否为"启用"、灰度比例是否 > 0 |
| 同义词匹配不准？ | 检查同义词优先级，调整优先级或增加更多同义词 |
| 会话过期时间多久？ | 默认 30 分钟无活动自动过期，可联系管理员调整 |
| 如何导出日志？ | 日志审计 → 筛选时间范围 → 导出 CSV |
| AI 生成规则可以信任吗？ | AI 规则作为建议，必须人工审核后才能上线 |
