# XCDOS / Prolog 数据合规清单与数据主体权利 SOP V1.0

> 版本：V1.0 ｜ 日期：2026-06-12 ｜ 关闭评审项：P0-12
> 性质：工程侧数据梳理。**正式合规结论须法务/安全负责人签署**（PIPL + 生成式 AI 服务管理暂行办法）。

## 一、数据清单（Data Inventory）

| 系统 | 数据类别 | 字段/载体 | 处理目的 | 合法性基础 | 保留期限 | 删除/匿名化方式 |
|---|---|---|---|---|---|---|
| 双系统 | 账号标识 | users.email / sys_user.username | 登录认证 | 合同履行 | 账号注销后 30 天 | 物理删除 + 审计留痕 |
| 双系统 | 口令 | password_hash（Argon2id） | 认证 | 合同履行 | 同账号 | 随账号删除 |
| 双系统 | 手机号 | users.phone / contact_phone | 通知/找回 | 同意 | 账号有效期 | 展示脱敏 + 删除时清除 |
| XCDOS | 业务正文 | feedbacks.* / goals / tasks | 核心业务 | 合同履行 | 永久（逻辑删除）/反馈 2 年 | 逻辑删除；删除请求走匿名化 |
| Prolog | 请求日志 | request_log（含 request_content/response_content/request_ip） | 排障/分析/自进化样本 | 正当利益 + 同意（样本） | 热 7 天 + 冷 90 天 | 到期物理清理 |
| Prolog | 训练样本 | request_log.is_sample=1 | 规则自进化 | **单独告知同意** | 跟随样本集版本 | 退出即移出样本池 |
| 双系统 | 操作审计 | 操作日志/审计事件 | 合规审计 | 法定义务 | ≥ 法定期限 | 不可删除（合规豁免） |
| 双系统 | IP / 设备 | request_ip / user_ip | 安全风控 | 正当利益 | 90 天 | 到期清理 |

> 脱敏原则：展示脱敏（手机号掩码）≠ 存储加密。敏感字段存储侧按需加密，展示侧掩码（修正评审 P2-08 语义混淆）。

## 二、数据主体权利（DSR）SOP

受理渠道：产品内「隐私请求」入口 + 隐私邮箱。身份核验：账号二要素。时限：**15 个工作日**（复杂可延长一次并告知）。

| 权利 | 流程 | 与「永久保留仅逻辑删除」冲突化解 |
|---|---|---|
| 查阅/复制 | 核验 → 导出该主体全量数据 JSON → 加密交付 | 无冲突 |
| 更正 | 核验 → 业务数据更正；反馈正文走 revision 不覆盖原文（P0-04） | 不可篡改与更正并存：新增修订版 |
| 删除 | 核验 → 账号与可删数据物理删除；业务核心数据**匿名化替代物理删除**（断开主体标识，保留统计） | 法定保留/审计数据豁免删除，向主体说明依据 |
| 撤回同意 | 关闭对应处理（如样本采集）；不影响已合法处理 | request_log.is_sample 置 0，移出样本池 |
| 导出（可携带） | 结构化机器可读格式导出 | 无冲突 |
| 拒绝自动化决策 | Agent 生成规则/评估提供人工复核通道（F-020 人工审核流） | 无冲突 |

## 三、训练样本授权

- request_log.is_sample=1 进入自进化样本池前，须经**显式告知同意**（注册条款 + 场景内提示）。
- 提供退出机制：撤回后样本移出池，已训练规则不溯及但停止继续采集。
- 样本去标识：进入训练前剥离 tenant_id 外的直接标识。

## 四、生成式 AI 合规（Prolog + Langflow）

- 面向公众的生成式能力须符合《生成式人工智能服务管理暂行办法》：语料合法性、内容标识、投诉举报入口、用户输入记录保护。
- Langflow 嵌入子系统：不得将租户业务数据用于跨租户模型训练；提示词与输出按请求日志策略留存与清理。
