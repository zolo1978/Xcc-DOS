XCDOS · Frontend UI Design Specification

深蓝三色专业版
前端 UI 设计规范

本规范适用于芯诚企业决策与执行操作系统 XCDOS 的 Web 后台界面，覆盖老板驾驶舱、拆推评算中心、HR Agent、PMC Agent、Sales Agent、Lean Agent 等页面。

一、设计原则

本系统不是活动页，也不是炫技大屏。它是企业决策与执行后台，第一优先级是稳定、清晰、可信、可扩展。

专业可信

以深蓝建立企业级、管理级、决策级系统气质。

信息优先

数据、状态、任务优先于装饰。

层级清晰

通过字号、间距、卡片、分区建立阅读路径。

三色克制

全局仅使用深蓝、蓝灰、青蓝三类主视觉颜色。

设计关键词

企业级深蓝克制高密度数据驱动Agent 协同

二、颜色系统

颜色不超过三类主视觉体系：深蓝主色、蓝灰中性色、青蓝强调色。状态也尽量通过标签文案和层级表达，避免红橙绿满天飞。

六、数据展示规范

数据展示不是堆数字，是让老板、管理层、员工快速判断：现在怎么样、哪里异常、下一步怎么办。

6.3 图表规范

  • 柱状图、折线图主色使用 Navy 700 → Cyan 500 渐变。
  • 背景网格线使用 BlueGray 100 / 200。
  • 不要使用超过 3 种图表颜色。
  • 图表必须给出单位、时间范围、数据口径。

七、状态与反馈规范

XCDOS 的状态重点不是“好看”，而是帮助组织及时纠偏。

状态展示方式交互建议
正常青蓝标签 + 正常文案可查看详情
待确认蓝灰标签 + 待确认文案提醒责任人确认
需老板确认深蓝强调点 + 标签文案进入老板驾驶舱顶部异常
已归档蓝灰弱标签默认折叠

反馈机制

  • 所有反馈必须包含：今日目标、今日结果、当前卡点、下一步动作。
  • 未反馈状态不建议只用红色警告，应进入“提醒 → 升级 → 追责”链路。
  • 管理层和一线员工使用同一反馈结构，不搞双重标准。

八、页面模板规范

以下页面是 XCDOS 的基础页面族，后续开发不得随意变体。

老板驾驶舱

经营总览、异常列表、Agent 协同图、重点决策入口。

拆推评算中心

问题输入、变量拆解、方案推演、可行性评分、ROI 测算。

目标管理

公司目标、部门目标、岗位目标、日任务拆解。

过程追踪

任务、节点、延期、异常、责任人、依赖关系。

强制反馈

日报、周报、月报、管理层汇报、异常升级。

Agent 模块页

HR、PMC、Sales、Production、Lean 各业务 Agent 工作台。

十、交付检查清单

每个前端页面交付前都按此检查，减少返工。

应该做到

  • 全局颜色严格控制在深蓝、蓝灰、青蓝三类体系内。
  • 页面背景使用 BlueGray 50，卡片使用白底。
  • 主按钮统一 Navy 800。
  • 数据卡片统一：指标名 + 核心数值 + 辅助说明。
  • 所有页面保持左侧导航、顶部栏、内容区三段式布局。

不要做

  • 不要引入红、橙、绿、紫等额外高饱和状态色。
  • 不要使用大面积复杂渐变背景。
  • 不要每个业务模块单独设计一套组件。
  • 不要为了“科技感”牺牲可读性。
  • 不要把报表页做成炫酷大屏，老板最终还是要看数。
