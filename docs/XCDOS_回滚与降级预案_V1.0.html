等级定义响应时效告警接收人示例

二、降级策略配置

2.1 性能降级

场景触发条件降级措施恢复条件
API 响应 P95 > 3s持续 5 分钟关闭 Agent 辅助功能，驾驶舱 KPI 缓存 TTL 延长至 15minP95 < 1s 持续 10 分钟
数据库连接池耗尽活跃连接 > 80%只读接口返回缓存数据，暂停写操作排队连接 < 50%
Redis 内存 > 80%持续 5 分钟非关键缓存逐出，缩短部分缓存 TTL内存 < 60%

2.2 功能降级

功能降级开关降级后行为用户提示
Agent 辅助拆解/评估FEATURE_AGENT_ASSIST隐藏 Agent 按钮，仅支持人工拆解"AI 辅助功能维护中，请手动操作"
Agent 反馈质量评分FEATURE_FB_QUALITY反馈提交后跳过 AI 评分无感知（静默）
决策报告导出FEATURE_REPORT_EXPORT报告仅支持在线查看，不支持下载"导出功能暂不可用"
实时通知推送FEATURE_PUSH_NOTIFY通知仅写入站内信，不推 IM/邮件无感知
全文搜索FEATURE_FULLTEXT_SEARCH降级为基础 LIKE 查询"搜索功能性能下降"
功能开关通过环境变量 / 配置中心控制，无需重启服务。开关变更 30 秒内生效。

三、回滚操作步骤

3.1 应用代码回滚

步骤操作执行人预计耗时
1确认回滚目标版本号（上一个稳定 tag）值班 SRE2 min
2通知团队：回滚原因、目标版本、预计影响时间技术负责人3 min
3灰度环境先回滚验证：kubectl set image deployment/xcdos xcdos={prev_tag}SRE5 min
4验证灰度环境功能正常（登录/驾驶舱/反馈提交）SRE + 测试5 min
5生产环境滚动回滚（逐个 Pod 替换）SRE10 min
6回滚后验证：核心接口 200、KPI 数据正常SRE5 min
总计~30 min

3.2 数据库回滚

场景回滚方式风险
DDL 变更（加字段）执行反向 ALTER TABLE DROP COLUMN数据丢失：DROP COLUMN 前备份当前表
DML 误操作（批量 UPDATE/DELETE）从最近全量备份 + WAL 日志恢复至误操作前时间点恢复期间服务不可用 ~15min
数据迁移脚本错误从迁移前快照恢复迁移后新写入数据丢失（先锁写再恢复）
数据库回滚前必须先锁写（SET session_replication_role = 'replica'），防止回滚期间新数据写入。
