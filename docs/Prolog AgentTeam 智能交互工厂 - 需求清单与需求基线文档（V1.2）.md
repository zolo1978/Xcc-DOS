### 3.2 增强功能

### 3.3 不做范围（Out of Scope）

- 不做实时协同编辑
- 不做在线 IDE（规则编辑器为纯文本编辑，不做语法高亮/自动补全）
- 不做 AI 模型自训练（仅做规则层面的聚类和生成）
- 不做跨租户数据联邦查询
- 不做移动端原生 App（仅 Web 管理端 + H5 会话端）

## 六、需求基线锁定

**基线锁定日期**：V1.2 需求评审通过后

**锁定原则**：
1. 需求基线一旦锁定，任何功能性新增/修改须走变更流程（CR）
2. P0 功能变更须技术负责人 + 产品负责人双审批
3. P1/P2 功能变更由产品负责人审批
4. 所有变更必须更新本文档并留痕

**基线锁定后允许的直接调整（不走变更）**：
- UI 文案优化（不改变交互逻辑）
- 错误提示语调整
- 非功能性的性能优化
