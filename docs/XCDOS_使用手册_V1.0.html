一、老板 / 高管操作指南

你怎么做：每天打开驾驶舱，看 4 个核心指标，异常立即下钻。不需要手动创建任何东西。

1.1 看懂驾驶舱

目标完成率

全公司目标的完成百分比。绿色=正常，黄色=滞后，红色=严重滞后。

任务执行率

已分配任务的完成比例。低于 60% 意味着大部分事情没人做。

反馈及时率

员工按时提交每日反馈的比例。低于 60% 意味着没人汇报进度。

1.2 处理异常

  1. 驾驶舱红色高亮的条目是最需要你关注的问题
  2. 点击异常条目 → 跳转异常详情，查看关联的目标、处理人、当前状态
  3. 如需干预：在异常详情中 @相关负责人，或直接分配处理人

1.3 下钻查看

点击任意 KPI 卡片可下钻到具体数据：点击"目标完成率"→ 查看按部门的目标完成情况；点击"异常数"→ 查看当前所有未解决异常。

二、管理者操作指南

你是系统的主要使用者。你的日常：创建目标 → 拆解问题 → 推演决策 → 分配任务。

2.1 创建目标

  1. 进入 /goals → 点击"新建目标"
  2. 填写：名称（如"Q3 销售额提升 20%"）、指标类型（选择 amount）、目标值（如 2000000）、期限
  3. 指定负责人 — 谁为这个目标负责
  4. 提交 → 目标进入草稿状态，可激活开始执行

2.2 目标拆解

  1. 在目标详情页点击"拆解子目标"
  2. 为每个子部门/子方向创建子目标
  3. 子目标自动关联父目标，形成目标树

2.3 使用拆推评算中心

这是 XCDOS 的核心。当你发现目标无法达成或出现问题时使用。

拆解阶段

  1. 从目标关联一个问题（如"销售额下滑 15%"）
  2. 列出可能的假设（如"竞品降价"、"销售团队流失"、"市场萎缩"）
  3. 为每个假设打分：证据强度 0-100、置信度 0-100

推演阶段

  1. 选择你倾向的假设方向
  2. 系统（或 Agent）生成推演方案

评估阶段

  1. 对方案进行 4 维评分：资源（需要多少人钱）、时间（多久见效）、风险（失败概率）、可行性（能做吗）

试算阶段

  1. 输入预估成本和预期收入
  2. 系统自动计算 ROI 和回收期

报告阶段

  1. 点击"生成报告"→ 自动汇总全流程 → 生成可分享的决策报告

2.4 分配任务

  1. 决策方案通过后 → 创建执行方案 → 拆分为具体任务
  2. 每个任务指定负责人和截止时间
  3. 负责人会收到通知，开始执行
最佳实践：不要跳过拆解直接做决策。拆解得越细，决策越靠谱。每个假设都要找反例——"什么情况下这个假设是错的？"

三、员工操作指南

你的核心动作只有两个：执行任务 + 每日反馈。

3.1 查看和更新任务

  1. 进入 /tasks 查看分配给你的任务列表
  2. 点击"开始执行" → 任务状态变为"进行中"
  3. 完成任务后点击"标记完成"

3.2 提交每日反馈

  1. 进入 /feedback → 选择关联的任务
  2. 填写 4 个字段：
    • 今日目标：你今天计划完成什么
    • 今日结果：你实际完成了什么
    • 卡点：什么阻碍了你（没有就填"无"）
    • 下一步：明天计划做什么
  3. 提交
每天 18:00 前提交反馈。未提交会收到系统提醒。反馈内容越具体越好——老板和 AI 都在看。

3.3 被分配异常时

  1. 收到异常通知 → 进入异常详情
  2. 确认问题 → 开始处理 → 更新状态为"处理中"
  3. 解决后标记"已解决"

四、系统管理员操作指南

4.1 管理用户与组织

  • 创建/编辑/禁用用户账号
  • 管理组织架构树
  • 分配用户角色（boss/admin/manager/employee）

4.2 监控 Agent Run

  • 进入 /agent-runs 查看所有 Agent 执行记录
  • 点击某条记录查看完整追踪（输入/输出/工具调用/阶段耗时）

4.3 审计日志

  • 查看用户操作记录：谁、在什么时候、做了什么操作
  • 可按人/资源类型/操作类型筛选

五、常见问题

问题答案
忘记提交反馈怎么办？次日可补交，但会标记为"延迟提交"影响及时率。建议设置每日 17:00 闹钟。
任务可以分配给多人吗？当前 MVP 一个任务只有一个负责人。如需多人协作，请将大任务拆分为多个子任务。
决策报告可以分享给谁？报告默认对同组织 manager+ 角色可见。可通过导出功能分享链接。
Agent 辅助生成的方案靠谱吗？Agent 方案作为参考和建议，不是最终决策。所有 Agent 输出都标注了置信度，建议人工审核后采纳。
驾驶舱数据多久更新一次？每 5 分钟刷新一次。关键事件（反馈提交、任务完成）触发实时更新。
